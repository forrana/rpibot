# Current Project

## What we are building
Python and arduino based project for remotely controlled RPI 5 based robot


## What good looks like
Test driven development, tests should come first
Clean and DRY code that uses abstractions and follows patterns where possible 
All code should be split in modules and follow single responsibility principle
Code should be easily readable and split into small 10-20 lines methods

## What to avoid
Putting everything in one file
Putting everything in one folder
Unnecessary comments
Commenting depricated code or renaming files to _old
